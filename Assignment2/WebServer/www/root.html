You can make the following GET requests:

  • /file/sample.html, which returns the content of the file sample.html
  • /json, which returns a JSON of the /random request.
  • /random, which returns index.html.

Requests with Arguments

The requests below require arguments to function. They include robust error handling.

Request Description Example URL (Copy & Paste)
/multiply? Multiplies two integers (num1 and num2). Handles missing or non-integer inputs (400 Bad Request). /multiply?num1=15&num2=3
/github? Fetches data from the GitHub API using a query parameter (e.g., repository list). Parses the JSON to display repository **name, ID, and owner login**. /github?query=users/amehlhase316/repos
/jsonquery? Parses a URL-encoded JSON string (data) and returns the value associated with a specified key. (Args: data, key) /jsonquery?data=%7B%22item%22%3A%22Widget%22%2C%22price%22%3A19.99%7D&key=price
/revstring? Reverses the string provided in text, but only starts the reversal after the character index specified by offset (integer). (Args: text, offset) /revstring?text=Engineer&offset=3
